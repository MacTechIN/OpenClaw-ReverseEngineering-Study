# [OpenClaw 연재 3회] 우리 프로젝트의 뼈대와 심장, 기초 공사 시작하기!

안녕하세요! OpenClaw(오픈클로) 뒤집어보기 시리즈, 벌써 **3회**차입니다. 

지난 시간에는 주방에 장비를 들여놓듯 Node.js, pnpm, TypeScript를 설치해 보았죠? 오늘은 본격적으로 OpenClaw라는 거대한 건물의 **'땅을 다지고 설계도를 펼치는'** 기초 공사 단계를 함께 진행해 보겠습니다.

---

## 1. 프로젝트의 거대한 트렁크, '모노레포(Monorepo)'

OpenClaw는 **모노레포** 구조를 선택했습니다.

*   **비유하자면?** 온 가족의 짐을 하나의 거대한 트렁크에 담는 것과 같습니다. 각자 작은 가방을 들고 다니면 잃어버리기 쉽지만, 하나로 묶어 관리하면 필요한 물건을 찾기도, 통째로 옮기기도 훨씬 쉬워지죠.

---

## 1-1. 프로젝트의 뼈대 살펴보기 (openclaw-src)

공식 소스 코드를 내려받아(Git Clone) 그 구조를 해부해 보면, OpenClaw가 얼마나 체계적으로 설계되었는지 알 수 있습니다.

### 🗺️ OpenClaw 초정밀 지도
```text
openclaw-src/ (프로젝트 루트)
├── 📂 apps/                # 실행 가능한 애플리케이션 (게이트웨이 등)
├── 📂 packages/            # 여러 곳에서 공용으로 쓰는 핵심 라이브러리
└── 📂 src/                 # 실제 모든 마법이 일어나는 핵심 소스 (90% 비중)
    ├── 📂 gateway/         # 🧠 [중추 신경망] 모든 메시지의 수신 및 배달 로직
    ├── 📂 channels/        # 🔌 [통신 포트] 외부 세상(디스코드, 슬랙 등)과의 입구
    ├── 📂 agents/          # 🤖 [AI 두뇌] 서버에 지능을 부여하는 핵심 로직
    ├── 📂 infra/           # 🧱 [바닥 기초] DB, Redis, 로그 시스템 등
    ├── 📂 types/           # 📏 [표준 규격] 프로젝트 전체 데이터 약속(Interface)
    └── 📂 utils/           # 🛠️ [공구함] 실무에서 자주 쓰는 유용한 함수들
```

> **[TIP]** 우리가 가장 먼저 공략할 핵심 심장은 바로 `src/gateway/` 입니다!

## 1-2. 프로젝트의 '출생 신고서'와 '통역 가이드'

거대한 마천루를 지을 때는 벽돌 한 장 쌓기 전에 **'지침서'**가 필요합니다. 

### 📄 package.json: 출생 신고서
이 프로젝트의 이름, 버전, 그리고 "내가 어떤 외부 도구(의존성)들의 도움을 받을 것인지"를 정의합니다. 이 파일이 없으면 컴퓨터는 이 폴더가 그냥 '폴더'인지 '프로그램'인지 알 길이 없습니다.

### 📄 tsconfig.json: 통역사의 가이드북
우리는 **TypeScript**를 씁니다. 인간에게는 친절하지만 컴퓨터는 바로 이해하지 못하죠. 그래서 누군가 중간에서 번역(컴파일)을 해줘야 합니다. `tsconfig.json`은 통역사에게 전달하는 **'정밀 조각 지침서'**입니다.

---

## 1-3. [실습] tsconfig.json 설정: 새로운 나라의 헌법 제정

나만의 기초 공사를 시작할 때, 아래의 **'핵심 3개 조항'**은 반드시 기억하세요!

```json
{
  "compilerOptions": {
    "target": "ES2022",    // [제1조] 최신 문법 사용 (성능과 가독성 업!)
    "module": "NodeNext",  // [제2조] 최신 통신 규약 준수
    "strict": true         // [제3조] 법 집행의 엄격함 (버그 사전 차단)
  }
}
```

> **[IMPORTANT]** "대충 짜고 나중에 고치지 뭐"라는 생각은 금물! 처음부터 엄격하게(`strict: true`) 관리하는 것이 나중에 버그를 찾느라 밤을 새우는 것보다 100배 경제적입니다.

## 1-4. 🚀 [마무리] 공사 내용 저장하기: Git Commit & Push

설계도까지 완료했다면, 이제 작업 내용을 안전하게 기록해둘 차례입니다.

### 📸 비유: 스냅샷 찍기와 앨범 전송
- **Git Commit (스냅샷)**: 현재 공사 진행 상황을 사진으로 찍어 내 컴퓨터의 **'비밀 앨범'**에 저장하는 것입니다.
- **Git Push (앨범 전송)**: 내 컴퓨터의 앨범을 **'중앙 서버(GitHub)'**라는 거대한 금고에 통째로 복사해서 올리는 것입니다.

---

### ☁️ [준비] 1분 만에 GitHub 온라인 금고 분양받기

Git Push를 하려면 먼저 내 코드를 안전하게 받아줄 온라인 공간이 필요합니다.

1.  **GitHub 회원 가입**:
    - [GitHub.com](https://github.com)에 접속해서 **[Sign up]**을 누르세요. 
    - 이메일과 비밀번호를 정하고, 나만의 멋진 **닉네임(Username)**을 만듭니다. (전 세계 개발자들의 공용 여권이 생기는 순간입니다!)
2.  **새 저장소(Repository) 만들기**:
    - 로그인 후 오른쪽 상단 **[+]** 아이콘 -> **[New repository]**를 클릭합니다.
    - **Repository name**에 `openclaw-lab`을 입력하세요.
    - 공개 여부(Public/Private)를 선택하고 맨 아래 **[Create repository]**를 누르면 온라인 금고가 생성됩니다!
3.  **내 컴퓨터와 온라인 금고 연결하기**:
    - 이제 내 컴퓨터에 있는 프로젝트 폴더에게 "너의 온라인 주소는 여기야!"라고 알려줘야 합니다. 
    - 터미널에서 아래 명령어를 입력하세요 (주소의 '내아이디' 부분은 본인의 계정명으로 바꿉니다).
    ```bash
    git remote add origin https://github.com/내아이디/openclaw-lab.git
    ```

---

### 🏗️ [실습] 나의 첫 기초공사 버전 기록하기

이제 모든 준비가 끝났습니다! 아래 명령어를 차례대로 입력해서 내 성과를 온라인 금고에 안전하게 저장해 봅시다.

```bash
# 1. 상태 확인: 바뀐 파일들을 살펴봅니다.
git status

# 2. 재료 담기: 기록할 파일들을 장바구니에 담습니다.
git add .

# 3. 기록하기: "기초 공사 완료"라는 메시지와 함께 스냅샷을 찍습니다.
git commit -m "feat: 기초 공사 및 설계도 완료"

# 4. 전송하기: 내 컴퓨터의 공사 기록을 온라인 금고(GitHub)로 보냅니다.
git push origin main
```
